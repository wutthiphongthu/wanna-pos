# PPOS — สรุปฟีเจอร์ที่มีอยู่ตอนนี้

เอกสารนี้อธิบายสิ่งที่ **ใช้งานได้ในโค้ดปัจจุบัน** (อ้างอิงจาก `lib/` และการนำทางหลัก) ไม่ใช่ roadmap ล้วนๆ

**อัปเดต:** สะท้อนสถานะโปรเจกต์ณเวลาที่สร้างไฟล์นี้

---

## สถาปัตยกรรม / แหล่งข้อมูล

| หัวข้อ | สถานะ |
|--------|--------|
| **Firebase Auth** | ใช้เข้าสู่ระบบ (ลงทะเบียน `IAuthService` เป็น Firebase เสมอ — ดู `lib/core/di/injector.dart`) |
| **ข้อมูลธุรกิจหลัก** | โหมดปัจจุบัน: **SQLite (Floor) + offline-first** (`AppConfig.dataSource = sqlite` ใน `lib/core/config/app_data_source.dart`) |
| **ซิงก์ Firestore** | มี `SyncResumeListener` และ implementation Firebase สำหรับบาง repository (ใช้เมื่อสลับ `AppDataSource.firebase` หรือผ่าน SyncManager ตามที่ออกแบบในโค้ด) |
| **DI** | `get_it` + `injectable` |
| **State** | หลัก ๆ ใช้ `flutter_bloc` และ `Cubit` ในบางหน้า |

---

## การเข้าสู่ระบบและร้าน (`features/auth`)

- หน้า **Login** และ **สมัครสมาชิก** (`RegisterPage` — Firebase Auth + สร้าง `users/{uid}`)
- สถานะ **AuthWrapper** → หลังล็อกอินสำเร็จเข้า **MainAppShell** (ไม่ผ่านหน้าเลือกโหมดแบบแยก route ใน `MaterialApp` หลัก)
- สถานะ **สร้างร้าน** (`AuthNeedsStore` → `CreateStorePage`) เมื่อยังไม่มีร้าน
- บทบาทผู้ใช้: **owner** / **employee** (ใน `AuthAuthenticated`)
- **Logout** ผ่านเมนู (`LogoutMenu`)

> หมายเหตุ: มีไฟล์ `mode_selection_page.dart` (เลือกหน้าขาย / หลังบ้าน / แดชบอร์ด) แต่ flow หลักหลัง auth คือ shell แบบ drawer ไม่ได้ผูก named route `/pos` ฯลฯ ใน `main.dart`

---

## Shell หลัก (`features/shell`)

- **Drawer** (`POSMenuDrawer`) สลับ 5 โซนใน `IndexedStack`:
  1. หน้าขาย (POS)
  2. หลังบ้าน (Backend)
  3. แดชบอร์ด
  4. สต็อก
  5. CRM

## หน้าขาย POS (`features/sales`)

- แสดงสินค้า **กริด**, โหลดสินค้าที่ active
- **ค้นหาในแถบด้านบน**: กรองแบบ realtime จากชื่อสินค้า, รหัสสินค้า (`productCode`), บาร์โค้ด, ชื่อรอง, รหัสบาร์โค้ดกำหนดเอง — ทำงานคู่กับการเรียง **ปักหมุด** (แสดงรายการที่ปักไว้ก่อนในผลที่กรองแล้ว)
- **ปักหมุดสินค้า**: ปุ่มหมุดมุมการ์ด — สินค้าที่ปักหมุดจะเรียงไว้ด้านบนก่อนรายการอื่น (เก็บต่อร้านใน SharedPreferences ผ่าน `PinnedProductsService`)
- **ตะกร้า / สรุปออเดอร์** (`OrderSummary`): เพิ่ม/ลดจำนวน, ส่วนลดรายการ, ส่วนลดทั้งบิล
- **เลือกลูกค้า/สมาชิก** (dialog ตอนเข้าหน้า — ปิดได้เพื่อขายแบบไม่ระบุลูกค้า)
- **ชำระเงิน**: เงินสด / โอน / ผสม, กรอกจำนวนรับ, คำนวณเงินทอน, บันทึกบิล
- คำนวณ **คะแนนสะสม** จากยอด (เชื่อมกับ config สมาชิก/โปรโมชันในโค้ด POS bloc)
- บันทึกขายผ่าน **`ISalesRepository`** (SQLite เป็นค่าเริ่มต้น)

## แดชบอร์ด (`features/dashboard`)

- สรุปยอด **วันนี้** (ยอดรวม, จำนวนบิล, เฉลี่ย/บิล)
- **% เติบโต** เทียบเมื่อวาน
- รายการ **บิลล่าสุด 20 รายการ**
- แตะบิลดู **รายละเอียด** (รายการสินค้า, ช่องทางชำระ, รับเงิน, ทอน)
- **ยกเลิกบิล** (จาก dialog) แล้วรีโหลดข้อมูล
- ปุ่ม **รีเฟรช**
- แสดงชื่อร้านจาก auth

สิ่งที่ยังไม่ครบเชิง “analytics”: กราฟ, เลือกช่วงวันที่, top สินค้า, รายงาน export ฯลฯ

---

## สต็อก (`features/stock`)

- แท็บ: **สินค้าทั้งหมด** / **ภาพรวม** / **ใกล้หมด**
- **CRUD สินค้า** (รายการ, รายละเอียด, ฟอร์ม)
- ค้นหา/กรองตาม use case ที่มีใน `stock/usecases/`
- บริการ **นำเข้า/ส่งออก** (`ProductImportExportService`) จากหน้าสต็อกหลัก
- เชื่อมหมวดหมู่ตอนแก้ไขสินค้า

---

## หมวดหมู่ (`features/categories`)

- หน้า **จัดการหมวดหมู่**เต็มรูปแบบ
- **Dialog จัดการด่วน** (quick dialog) ใช้จากหลังบ้าน/สต็อก
- BLoC สำหรับ CRUD / สถานะหมวดหมู่

## หลังบ้าน (`features/backend`)

## การชำระเงินแยกเป็นโมดูล (`features/payment`)

- มี **entity โดเมน** (`payment.dart`) — **ยังไม่มีหน้า payment แยก**; flow ชำระเงินอยู่ในหน้าขาย (`OrderSummary` / `PosBloc`)

## อื่น ๆ ที่ควรรู้

- **Seed ข้อมูลตัวอย่าง**: `SampleData.seedProducts()` เมื่อไม่ใช้โหมด Firebase-only (`main.dart`)
- **ธีม**: Material / `AppTheme` (รองรับ light/dark ใน `main.dart`)

---

## สรุปสั้น ๆ

แอปมี **POS ครบ flow ขาย–ชำระ–บันทึก**, **สต็อก+หมวดหมู่**, **แดชบอร์ดสรุปยอด/บิล**, **CRM ส่วนสมาชิก+คะแนน**, **Auth+ร้าน** และโครง **offline-first + Auth Firebase** ตาม config ปัจจุบัน  
ส่วนที่ยังบางหรือเป็น placeholder ชัดเจนคือ **CRM ลีด/กิจกรรม/รายงาน**, **analytics แดชบอร์ดเชิงลึก**, และ **โมดูล payment แยกเป็นหน้า**
